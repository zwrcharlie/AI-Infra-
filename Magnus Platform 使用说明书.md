# Magnus Platform 使用说明书

> 这是一份面向新手的详细说明书，帮助你快速了解 Magnus Platform 的每一个功能。  
> 平台地址：<http://162.105.151.134:3011/>

## 一、Magnus Platform 是什么？

简单来说，Magnus Platform 是一个**团队协作 + AI 任务管理平台**。

你可以把它想象成一个“AI 工作台”：
- 你可以在上面**运行 AI 任务**（比如训练一个模型、跑一段代码）
- 你可以**管理团队成员和 AI 助手**
- 你可以和同事**聊天沟通**
- 你可以查看**服务器资源**（比如还有多少 GPU 可以用）

平台左侧有一个导航栏，包含 9 个功能模块，下面逐一介绍。

## 二、9 大功能模块详解

### 1. 启新（Explorer）—— AI 对话助手

**在哪里**：左侧导航栏点击「启新」

**这是什么**：类似 ChatGPT 的 AI 对话工具。你可以向 AI 提问，AI 会回答你。

**怎么用**：
1. 打开「启新」页面，你会看到一个输入框
2. 在输入框中输入你的问题或指令，比如“帮我分析这段数据”
3. 你还可以**粘贴图片**（按 Ctrl+V 粘贴剪贴板中的图片），AI 也能看到图片内容
4. 按回车发送，AI 会创建一个新的“会话”来回答你

**其他操作**：
- **重命名会话**：点击会话标题，直接修改名字
- **分享会话**：点击分享按钮，会生成一个链接，把链接发给别人，别人就能看到你和 AI 的对话
- **取消分享**：再次点击分享按钮，关闭分享
- **删除会话**：不需要的会话可以删除
- **上传文件**：可以上传文件让 AI 分析

### 2. 消息（Chat）—— 聊天功能

**在哪里**：左侧导航栏点击「消息」

**这是什么**：和微信聊天类似的即时通讯功能，可以和同事或 AI 助手聊天。

**怎么用**：

**发起私聊（一对一聊天）**
1. 点击「消息」页面右上角的「新建会话」按钮
2. 选择「私聊」类型
3. 从成员列表中选择你想聊天的人
4. 点击创建，就可以开始聊天了

**发起群聊**
1. 点击「新建会话」按钮
2. 选择「群聊」类型
3. 给群聊起一个名字
4. 选择多个成员
5. 点击创建

**其他操作**
- 左侧会显示你的所有会话列表，最新消息的会话排在最上面
- 支持发送**图片消息**
- 只有会话的创建者才能删除会话

### 3. 人事（People）—— 团队管理

**在哪里**：左侧导航栏点击「人事」

**这是什么**：管理团队中所有人员（包括真人和 AI 助手）的地方。

**怎么用**：

**查看团队成员**
- 打开页面就能看到所有成员的列表
- 可以用搜索框搜索特定的人
- 每个人会显示：名字、角色（管理员/普通成员）、上级是谁、管理的蓝图/服务/技能数量

**招募 AI 助手（Agent）**
1. 点击「招募」按钮
2. 输入 AI 助手的名字
3. 可以上传一个头像
4. 点击提交，一个新的 AI 助手就创建好了

**管理 API Token（密钥）**
- 点击某个成员，打开详情面板
- 可以查看、复制、刷新该成员的 API Token
- Token 格式长这样：`sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
- Token 相当于“密码”，用于程序调用平台的接口

**管理编制（Headcount）**
- 编制就是一个成员能同时运行多少个任务
- 管理员可以修改成员的编制数

**快捷操作**
- **发消息**：点击成员旁边的消息图标，直接和他私聊
- **邀请到群聊**：可以把一个成员加入已有的群聊

### 4. 任务（Jobs）—— 运行和管理计算任务

**在哪里**：左侧导航栏点击「任务」

**这是什么**：管理你提交到服务器上运行的任务。比如你要训练一个 AI 模型，就需要创建一个“任务”。

**怎么用**：

**创建新任务**
1. 点击「新建任务」按钮
2. 填写任务配置（运行什么代码、需要多少资源等）
3. 提交后任务进入排队

**任务的生命周期（状态变化）**  
一个任务会经历这些阶段：

> 排队中(Pending) → 准备中(Preparing) → 运行中(Running) → 完成/失败

- **排队中**：等待服务器资源空闲
- **准备中**：正在准备运行环境（比如下载代码、拉取镜像）
- **运行中**：任务正在执行
- **完成**：任务执行结束

**其他操作**
- **搜索任务**：在搜索框输入关键词
- **按用户筛选**：下拉菜单选择某个用户，只看他的任务
- **克隆任务**：复制一个已有任务的配置，快速创建新任务
- **终止任务**：强制停止正在运行的任务
- 页面会**自动刷新**，实时显示任务状态

### 5. 蓝图（Blueprints）—— 任务模板

**在哪里**：左侧导航栏点击「蓝图」

**这是什么**：蓝图就是**任务模板**。如果你经常运行类似的任务（比如每次训练模型只是改几个参数），就可以把任务做成蓝图，下次直接用。

**打个比方**：蓝图就像是一个“食谱”，里面写好了做菜的步骤，你只需要选择放多少盐、多少糖（参数），就能做出一道菜（运行一个任务）。

**怎么用**：

**创建蓝图**
1. 点击「新建」按钮
2. 填写蓝图名称和描述
3. 在代码编辑器中编写蓝图代码（定义任务怎么运行、有哪些可配置的参数）
4. 保存

**运行蓝图**
1. 在蓝图列表中找到你要运行的蓝图
2. 点击「运行」按钮
3. 平台会弹出一个表单，显示这个蓝图的所有可配置参数
4. 填写参数（如果之前运行过，会自动填充上次的参数）
5. 点击「启动」，一个新任务就创建好了
6. 页面会自动跳转到「任务」页面，你可以看到新任务的运行状态

**其他操作**
- **克隆蓝图**：复制一个已有蓝图，在此基础上修改
- **删除蓝图**：需要输入蓝图 ID 确认，防止误删
- **转移所有权**：把蓝图的管理权交给其他人

### 6. 服务（Services）—— 长期运行的程序

**在哪里**：左侧导航栏点击「服务」

**这是什么**：服务是**一直运行着的程序**，和“任务”不同——任务跑完就结束了，而服务会持续运行。

**打个比方**：任务像是“洗一次衣服”，洗完就停了；服务像是“开着的洗衣店”，一直营业，随时可以接活。比如一个 AI 对话接口服务，就需要一直运行来响应用户请求。

**怎么用**：

**创建服务**
1. 点击「新建」按钮
2. 填写服务名称、描述和配置
3. 保存

**启动/停止服务**
- 创建好的服务默认是**未激活**的
- 点击「启动」按钮，服务开始运行（会在后台创建一个任务）
- 点击「停止」按钮，服务暂停运行
- 每次操作前都会弹出确认对话框，防止误操作

**查看服务状态**
- 服务列表会显示当前状态：活跃/非活跃
- 如果服务正在运行，会显示关联的任务状态（排队中/准备中/运行中/暂停）
- 点击状态标签可以跳转到对应的任务详情

**其他操作**
- **排序**：按最后活跃时间 或 最后更新时间 排序
- **筛选**：按拥有者筛选，或只看活跃的服务
- **克隆/删除/转移所有权**：和蓝图类似

### 7. 技能（Skills）—— 代码包管理

**在哪里**：左侧导航栏点击「技能」

**这是什么**：技能是**可复用的代码包**。一个技能包含多个文件，可以被蓝图或服务引用。

**打个比方**：如果蓝图是“食谱”，那技能就是“预制的调料包”。你可以在不同的食谱中使用同一包调料，不用每次都从头配。

**怎么用**：

**创建技能**
1. 点击「新建」按钮
2. 填写技能名称和描述
3. 编写代码文件（一个技能可以包含多个文件，每个文件有路径和内容）
4. 保存

**其他操作**
- **克隆**：复制已有技能进行修改
- **删除**：需要确认，防止误删
- **转移所有权**：将技能交给其他人管理

### 8. 镜像（Images）—— 运行环境管理

**在哪里**：左侧导航栏点击「镜像」

**这是什么**：镜像是任务运行所需的**软件环境**。

**打个比方**：你在自己电脑上写代码需要先安装 Python、PyTorch 等软件。镜像就是把这些软件**提前打包好**的环境，任务启动时直接用，不需要每次都装一遍。

**怎么用**：

**预热（注册）镜像**
1. 点击「预热」按钮
2. 输入镜像的地址（URI），比如：`docker://pytorch/pytorch:2.5.1-cuda12.4-cudnn9-runtime`
3. 点击提交，平台会开始下载这个镜像

**镜像状态说明**

| 状态 | 含义 |
|------|------|
| cached（已缓存） | 镜像已下载完成，可以直接使用 |
| pulling（拉取中） | 正在从远程仓库下载镜像 |
| refreshing（刷新中） | 正在重新下载最新版本 |
| missing（缺失） | 镜像找不到了，可能被删除了 |

**其他操作**
- **刷新**：重新下载镜像的最新版本
- **删除**：移除不再需要的镜像
- **查看详情**：点击镜像可以看到大小、拥有者、创建时间等信息

### 9. 集群（Cluster）—— 资源监控面板

**在哪里**：左侧导航栏点击「集群」

**这是什么**：显示服务器当前资源使用情况的**仪表盘**。

**打个比方**：就像汽车仪表盘告诉你油量、速度一样，集群面板告诉你服务器还有多少资源可用。

**怎么用**：

页面会显示四个关键指标：

| 指标 | 含义 | 例子 |
|------|------|------|
| 可用 CPU + 内存 | 服务器还有多少计算核心和内存 | 空闲 32 核 / 总共 128 核，空闲 64GB / 总共 256GB |
| 可用 GPU | 还有多少显卡可以用 | 空闲 4 张 / 总共 8 张 A100 |
| 活跃任务数 | 当前正在运行多少个任务 | 12 个任务运行中 |
| 队列深度 | 有多少任务在排队等待 | 5 个任务排队中 |

页面下方还有三个任务列表：
- **我的活跃任务**：你自己正在运行的任务
- **运行中任务**：所有人正在运行的任务
- **排队中任务**：等待资源的任务

页面会**自动刷新**，实时更新数据。

## 四、常见操作流程

> **流程一：我想跑一个 AI 训练任务**
> 1. 先去「镜像」页面，确认需要的运行环境镜像已经预热好了
> 2. 去「集群」页面，看看还有没有空闲的 GPU
> 3. 去「任务」页面，点击「新建任务」
> 4. 选择镜像、填写代码和资源需求、提交
> 5. 在「任务」页面等待任务从“排队”变成“运行中”
> 6. 任务完成后查看结果

> **流程二：我经常要跑类似的任务，每次只改几个参数**
> 1. 去「蓝图」页面，创建一个蓝图模板
> 2. 在蓝图代码中定义好可配置的参数
> 3. 以后每次运行时，直接点蓝图的「运行」按钮
> 4. 在弹出的表单中填写这次的参数值
> 5. 点击启动，任务自动创建

> **流程三：我想部署一个一直运行的 AI 服务**
> 1. 先准备好镜像和技能（代码包）
> 2. 去「服务」页面，创建一个新服务
> 3. 配置好后，点击「启动」
> 4. 服务会在后台一直运行
> 5. 需要更新时，先停止再修改，然后重新启动

> **流程四：我想和同事讨论任务结果**
> 1. 去「消息」页面
> 2. 发起私聊或群聊
> 3. 直接在聊天中交流，还可以发图片

## 五、小贴士

1. **不确定有没有资源？** 先看「集群」页面再提交任务
2. **任务运行失败？** 检查镜像是否已缓存、资源需求是否超过可用量
3. **想重跑上次的任务？** 用「克隆」功能一键复制
4. **蓝图参数记不住？** 平台会自动记住你上次填的参数
5. **页面显示“正在验证权限”？** 说明你还没登录，等待自动登录或点击飞书登录按钮
6. **Token 是什么？** 它是调用平台 API 的密钥，不要分享给不相关的人
